### [Andrej Karpathy talks about "Claws"](https://simonwillison.net/2026/Feb/21/claws/#atom-everything) ⭐️ 8.0/10

Andrej Karpathy, a leading AI researcher, publicly discussed the emerging concept of 'Claws,' describing them as a new orchestration layer built on top of existing LLM agents. He specifically highlighted projects like NanoClaw, which features a core engine of about 4000 lines of code and runs agents in containers by default. This matters because it signals an architectural evolution in AI systems, where 'Claws' could become a standard layer for managing complex agent workflows, improving scheduling, tool use, and persistence. If widely adopted, this layer could make AI agents more reliable, secure, and manageable for both personal and professional use. A key technical detail is that NanoClaw's design prioritizes a small, auditable codebase and containerized execution for improved security, contrasting with the potentially vulnerable architecture of existing systems like OpenClaw. The ecosystem is already diversifying with multiple implementations like zeroclaw, ironclaw, and picoclaw, indicating rapid community experimentation around this core concept.

*Source: rss/Simon Willison*

**Background**: LLM agents are AI systems that use large language models to perform tasks by breaking them down, planning steps, and using tools or APIs. An orchestration layer is software that manages and coordinates these agents, handling complex workflows, memory, and tool calls. The discussion references OpenClaw as an early, popular system in this category, which has reportedly seen over 150,000 instances deployed, primarily by private users.

### [Taalas serves Llama 3.1 8B at 17,000 tokens/second](https://simonwillison.net/2026/Feb/20/taalas/#atom-everything) ⭐️ 8.0/10

Taalas, a Canadian hardware startup, announced its first product, a custom hardware chip called Silicon Llama that runs Meta's Llama 3.1 8B model at 17,000 tokens per second. This performance is achieved through an aggressive quantization technique that combines 3-bit and 6-bit parameters. This represents a potential paradigm shift in efficient AI deployment, as such extreme inference speeds could enable real-time, high-volume AI applications previously limited by computational bottlenecks. It demonstrates a new approach where models are hard-wired into custom silicon, which could significantly lower the cost and energy consumption of running large language models at scale. The company states its current generation uses a mix of 3-bit and 6-bit quantization, and its next generation will move to 4-bit parameters, indicating an ongoing hardware development cycle. The product is available for a live demo via chatjimmy.ai, allowing users to experience the speed firsthand.

*Source: rss/Simon Willison*

**Background**: Quantization is a technique to reduce the precision of numbers in a neural network, typically from 32-bit or 16-bit floating-point values to lower bit-widths like 8-bit or 4-bit integers, which decreases the model's memory footprint and computational requirements. Llama 3.1 8B is a popular, open-source large language model with 8 billion parameters, released by Meta in July 2024. Custom AI silicon refers to specialized hardware chips designed from the ground up to run specific AI models or workloads more efficiently than general-purpose processors like CPUs or GPUs.

### [Quoting Thariq Shihipar](https://simonwillison.net/2026/Feb/20/thariq-shihipar/#atom-everything) ⭐️ 7.0/10

An Anthropic engineer, Thariq Shihipar, revealed that prompt caching is a foundational technique enabling the economic viability of Claude Code, a long-running AI agent product, by reusing computation from previous interactions to reduce latency and cost. The team monitors the prompt cache hit rate as a critical business metric and declares SEV (Severity) alerts if the rate drops too low, as it directly impacts their ability to offer generous rate limits in subscription plans. This insight matters because it reveals the crucial production engineering required to scale and commercialize AI agent products, showing that cost optimization techniques like caching are not just technical details but core business drivers that influence pricing and service reliability. It highlights a shift in focus from pure model capability to operational efficiency and cost management as key factors for sustainable AI products in the market. The entire Claude Code system harness is built around prompt caching, indicating it is a central architectural pillar rather than an optional optimization. The direct linkage between a high cache hit rate and the ability to set more generous user rate limits shows how technical performance metrics are tightly coupled to business model decisions in production AI systems.

*Source: rss/Simon Willison*

**Background**: Prompt caching is a technique for improving the efficiency of Large Language Model (LLM) queries by storing and reusing the computed results of frequently used prompts, which saves time and money. In transformer-based LLMs, caching often happens within the 'attention' mechanism, allowing the system to skip redundant computations. A 'cache hit rate' measures how often requested data is found in the cache versus needing a fresh computation. SEV (Severity) alerts are standardized incident levels used in production engineering to prioritize and respond to system issues, ranging from minor (SEV 5) to critical outages.

### [ggml.ai joins Hugging Face to ensure the long-term progress of Local AI](https://simonwillison.net/2026/Feb/20/ggmlai-joins-hugging-face/#atom-everything) ⭐️ 8.0/10

The organization ggml.ai, which is the founding team behind the llama.cpp project, has officially joined Hugging Face, as announced on February 20, 2026. The stated joint objectives include creating seamless integration with the Transformers library and improving the packaging and user experience of ggml-based software. This represents a significant consolidation in the open-source AI infrastructure space, potentially ensuring the long-term stewardship of key local AI tools. It could lead to model releases that are compatible with the GGML ecosystem out of the box, which would be a major win for the local model movement by simplifying deployment for casual users. A key technical goal is achieving 'single-click' integration between the Transformers library, which is the dominant framework for model definitions, and the ggml ecosystem for efficient local inference. The announcement specifically highlights a focus on improving the user experience for deploying local models, an area previously served by downstream projects like Ollama and LM Studio.

*Source: rss/Simon Willison*

**Background**: llama.cpp is a foundational open-source C/C++ library that enables running large language models (LLMs) locally on consumer hardware, famously starting with a hack to run a 4-bit quantized model on a MacBook. Its development is tied to the GGML project, a general-purpose tensor library designed for efficient inference. Before llama.cpp, running models like Meta's original LLaMA required PyTorch, specific GPU hardware (NVIDIA/CUDA), and was less accessible. Hugging Face's Transformers library is the de facto standard framework for using and releasing LLMs in the Python ecosystem.
