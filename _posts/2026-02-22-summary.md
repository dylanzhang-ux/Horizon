### [Andrej Karpathy talks about "Claws"](https://simonwillison.net/2026/Feb/21/claws/#atom-everything) ⭐️ 8.0/10

Andrej Karpathy, a leading AI researcher, publicly discussed the emerging concept of 'Claws' as a new orchestration layer above LLM agents, highlighting specific implementations like NanoClaw which features a minimal ~4000-line codebase and runs agents in containers by default. He identified this as an architectural evolution where Claws handle orchestration, scheduling, context, tool calls, and persistence for AI agents running on personal hardware. This matters because Karpathy's endorsement and terminology often shape industry discourse, and his framing positions Claws as a significant new layer in the AI stack that could standardize how complex, persistent agentic systems are built and managed. The emergence of minimalist alternatives like NanoClaw suggests a trend toward more auditable, secure, and developer-friendly frameworks for agent orchestration. A key technical detail is NanoClaw's architectural choice to run everything in isolated Linux containers by default, which directly addresses security concerns present in other frameworks like OpenClaw. Furthermore, its core engine of roughly 4000 lines of code is highlighted as being both human-comprehensible and AI-agent-comprehensible, promoting manageability and auditability.

*Source: rss/Simon Willison*

**Background**: LLM (Large Language Model) agents are AI systems that use language models to reason, plan, and execute actions using tools or APIs. 'Orchestration' in this context refers to the software layer that manages these agents, handling tasks like scheduling their runs, maintaining context between interactions, and managing their access to tools. The discussion assumes familiarity with the trend of running AI agents on personal hardware and the concept of 'agentic' systems that can act autonomously.

### [Taalas serves Llama 3.1 8B at 17,000 tokens/second](https://simonwillison.net/2026/Feb/20/taalas/#atom-everything) ⭐️ 8.0/10

Canadian hardware startup Taalas announced its first product, a custom hardware implementation of Meta's Llama 3.1 8B model that achieves an inference speed of 17,000 tokens per second. This performance is enabled by an "aggressively quantized" design combining 3-bit and 6-bit parameters. This represents a major breakthrough in efficient AI deployment, potentially enabling real-time, high-throughput applications like interactive chatbots or large-scale content generation that were previously impractical. It demonstrates how custom silicon and aggressive quantization can dramatically reduce the cost and latency of running large language models. The company's "Silicon Llama" uses a mixed-precision approach with 3-bit and 6-bit parameters, which is more aggressive than common 4-bit or 8-bit quantization techniques. Taalas indicates their next generation will use 4-bit quantization, suggesting they have a long hardware development cycle and that the current design involves significant trade-offs in model fidelity for speed.

*Source: rss/Simon Willison*

**Background**: Quantization is a technique that reduces the numerical precision of a model's parameters (e.g., from 16-bit to 4-bit), shrinking its memory footprint and computational requirements, which often comes at the cost of some accuracy. Llama 3.1 8B is a popular, open-source large language model released by Meta in July 2024. Hardware acceleration for LLM inference involves designing specialized chips (like GPUs, FPGAs, or ASICs) to run these models faster and more efficiently than general-purpose processors.

### [ggml.ai joins Hugging Face to ensure the long-term progress of Local AI](https://simonwillison.net/2026/Feb/20/ggmlai-joins-hugging-face/#atom-everything) ⭐️ 8.0/10

Georgi Gerganov's organization, ggml.ai, and its flagship project llama.cpp have officially joined Hugging Face, with the joint goal of advancing local AI capabilities through better integration with the Transformers library and improved user experience for ggml-based software. This represents a significant consolidation in the open-source AI infrastructure space, potentially accelerating the development and accessibility of local AI by combining the leading model inference library (llama.cpp) with the dominant model framework (Transformers). It signals a major investment in making local inference a more viable and user-friendly alternative to cloud-based AI services. The collaboration specifically aims for seamless 'single-click' integration between the Transformers library and the ggml ecosystem, which is crucial for wider model support and quality control, and will focus on improving the packaging and deployment experience for casual users of local models.

*Source: rss/Simon Willison*

**Background**: Local AI refers to running large language models (LLMs) directly on consumer hardware like laptops, rather than relying on cloud servers. The llama.cpp project, created by Georgi Gerganov in March 2023, was a breakthrough that made this possible by implementing LLM inference in pure C/C++ with no dependencies, enabling models to run on a wide range of hardware, including MacBooks, often using 4-bit quantization to reduce memory usage. Hugging Face's Transformers library is the de facto standard framework for defining and working with AI models in the open-source community.

### [How I use Claude Code: Separation of planning and execution](https://boristane.com/blog/how-i-use-claude-code/) ⭐️ 7.0/10

A developer published a detailed workflow for using Claude Code that advocates for a strict separation between a planning phase, where the AI analyzes code and creates a detailed implementation plan, and an execution phase, where it follows that plan to write the actual code. This structured approach directly addresses a core challenge in AI-assisted programming—preventing the model from making incorrect assumptions or skipping steps—which can significantly improve the reliability and quality of the generated code for developers. The workflow specifically instructs Claude to read code 'deeply' and in 'great detail' during planning, as surface-level reading leads to errors, and it emphasizes creating a concrete artifact like a markdown file or GitHub issue to document the plan before any code is written.

*Source: hackernews/vinhnx*

**Background**: Claude Code is an AI-powered coding assistant from Anthropic that helps with tasks like building features and fixing bugs. In AI-assisted programming, a common problem is that language models can hallucinate or make contextually wrong decisions if they don't fully understand the existing codebase before starting implementation.

**Discussion**: Commenters strongly validated the approach, noting it mirrors how senior engineers already work and is a more reliable pattern than fully autonomous 'agent' workflows, which require very specific validation tooling. One user suggested creating a GitHub issue instead of a markdown file for a permanent record, while another expressed curiosity about why explicitly telling the model to read 'deeply' affects its performance.

### [Quoting Thariq Shihipar](https://simonwillison.net/2026/Feb/20/thariq-shihipar/#atom-everything) ⭐️ 7.0/10

An Anthropic engineer, Thariq Shihipar, revealed that prompt caching is the foundational technique enabling the economic viability of their long-running Claude Code agent product, and that they monitor cache hit rates so closely that low rates trigger SEV (Severity) alerts. This insight matters because it shows how core infrastructure optimizations like caching directly influence the business model and user experience of AI products, determining their cost structure, the generosity of user rate limits, and the operational health of the service. The engineering team has built Claude Code's entire operational harness around prompt caching, and the cache hit rate is a critical business metric that directly impacts subscription plan rate limits and triggers high-priority incident responses when it falls too low.

*Source: rss/Simon Willison*

**Background**: Prompt caching is an optimization technique for Large Language Models (LLMs) where the computed results for identical or similar prompt prefixes are stored and reused, avoiding redundant computation to save both time and cost. In production software systems, an SEV (Severity) alert is a high-priority incident notification triggered when a key performance metric falls outside acceptable bounds, requiring immediate attention. Claude Code is an AI agent product from Anthropic designed for long-running, interactive coding assistance.
