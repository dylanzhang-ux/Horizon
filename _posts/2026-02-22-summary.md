### [Andrej Karpathy talks about "Claws"](https://simonwillison.net/2026/Feb/21/claws/#atom-everything) ⭐️ 8.0/10

Andrej Karpathy identified 'Claws' as a new orchestration layer built on top of LLM agents, highlighting specific implementations like NanoClaw which features a core engine of approximately 4,000 lines of code and runs everything in containers by default. He noted that multiple smaller Claws projects are emerging, including nanobot, zeroclaw, ironclaw, and picoclaw, with 'Claw' becoming a term for OpenClaw-like agent systems. This matters because it signals an architectural evolution in AI systems where Claws provide advanced orchestration, scheduling, context management, and persistence for LLM agents, potentially making agent systems more manageable, auditable, and flexible for developers. The emergence of this layer could give developers building these systems an early advantage as agent orchestration becomes more widespread. A key technical detail is that NanoClaw's architecture reduces core logic to roughly 500 lines of TypeScript according to other sources, making it auditable by a human or secondary AI in minutes, and it employs a single-process Node.js orchestrator with container isolation for security. The concept emphasizes systems that run on personal hardware, communicate via messaging protocols, and can both act on direct instructions and schedule tasks.

*Source: rss/Simon Willison*

**Background**: LLM agents are systems that use large language models to perform tasks by breaking them down, using tools, and making decisions. An orchestration layer manages how these agents work together, handling scheduling, context, and state between different tool calls and agent actions. OpenClaw is an open-source project that represents an early implementation of this 'Claw' concept for personal AI agents.

### [Taalas serves Llama 3.1 8B at 17,000 tokens/second](https://simonwillison.net/2026/Feb/20/taalas/#atom-everything) ⭐️ 8.0/10

Canadian hardware startup Taalas announced its first product, a custom hardware implementation of Meta's Llama 3.1 8B model that can run inference at 17,000 tokens per second. This performance is achieved through an approach they call 'Silicon Llama,' which involves hard-wiring the model into custom silicon. This represents a major breakthrough in efficient AI deployment, potentially making high-speed, low-latency LLM inference nearly 10 times faster than current state-of-the-art solutions and at a lower cost. If scalable, this technology could enable ubiquitous, real-time AI applications that were previously impractical due to performance or cost constraints. The speed is achieved through aggressive quantization, combining 3-bit and 6-bit parameters, which significantly reduces the model's memory footprint and computational requirements. The company has indicated their next generation will use 4-bit quantization, suggesting they have a long hardware development cycle for baking out new models.

*Source: rss/Simon Willison*

**Background**: Quantization is a technique that reduces the precision of a model's numerical parameters (like from 16-bit to 4-bit or lower) to decrease its memory usage and increase inference speed, often with a minor trade-off in accuracy. Hardware acceleration for LLM inference involves designing specialized chips (like GPUs, TPUs, or custom ASICs) that are optimized for the specific mathematical operations required by large language models. Llama 3.1 8B is a popular, open-source language model with 8 billion parameters released by Meta in July 2024, often used as a benchmark for efficient, locally-runnable AI.

### [Quoting Thariq Shihipar](https://simonwillison.net/2026/Feb/20/thariq-shihipar/#atom-everything) ⭐️ 7.0/10

An Anthropic engineer, Thariq Shihipar, revealed that prompt caching is a foundational technology enabling the economic viability of long-running AI agent products like Claude Code, and that the team actively monitors cache hit rates, declaring SEV (Severity) alerts when rates drop too low. The engineering team has built their entire system harness around this caching strategy to reuse computation from previous interactions. This insight matters because it shows how core infrastructure optimizations like prompt caching directly translate to business outcomes, including lower operational costs, reduced latency for users, and the ability to offer more generous API rate limits in subscription plans. It highlights that for production AI systems, engineering efficiency metrics are now critical business KPIs that can trigger high-priority incident responses. A key technical detail is that the prompt cache hit rate is so critical to the product's economics that a low rate is treated as a production incident warranting a SEV alert, which typically mobilizes engineers for an immediate fix. The system reuses computation from previous LLM roundtrips, which implies caching is applied to repeated or similar prompts within an agent's session.

*Source: rss/Simon Willison*

**Background**: Prompt caching is an optimization technique for Large Language Models (LLMs) where frequently unchanged parts of a prompt, like instructions or reference material, are stored and reused to avoid reprocessing the same tokens, significantly reducing cost and latency. SEV (Severity) alerts are a standard incident management classification in production systems, where a SEV1 or SEV2 indicates a critical issue requiring immediate attention, often outside normal working hours. Claude Code is an AI agent product from Anthropic designed to assist with programming tasks, and like other agents, it can involve long, multi-step interactions with an LLM.

### [ggml.ai joins Hugging Face to ensure the long-term progress of Local AI](https://simonwillison.net/2026/Feb/20/ggmlai-joins-hugging-face/#atom-everything) ⭐️ 8.0/10

The organization ggml.ai, which maintains the foundational ggml tensor library and the llama.cpp project, has officially joined Hugging Face as of February 2026, with the stated goal of ensuring the long-term progress of local AI. This move represents a strategic consolidation, bringing the key infrastructure for running large language models (LLMs) locally under the same umbrella as the industry-standard Transformers library. This matters because it consolidates two of the most critical pillars of the open-source AI ecosystem, potentially streamlining the process for developers to create and deploy models that work seamlessly both in the cloud and locally. It signals a major investment in improving the user experience and packaging for local AI, which could accelerate the adoption of local inference as a competitive alternative to cloud-based AI services. A primary technical objective of the merger is to achieve seamless 'single-click' integration between the Hugging Face Transformers library and the ggml ecosystem, which would allow models to be compatible with local inference tools like llama.cpp out of the box. Furthermore, the announcement emphasizes a commitment to improving the packaging and user experience of ggml-based software, an area previously served mostly by third-party tools like Ollama and LM Studio.

*Source: rss/Simon Willison*

**Background**: Local AI refers to running artificial intelligence models, like large language models, directly on a user's own hardware (like a laptop) instead of relying on cloud servers, offering benefits in data privacy, cost, and offline use. The ggml library, created by Georgi Gerganov, is a low-level tensor library written in C that enables efficient model execution on commodity hardware through techniques like 4-bit integer quantization. Llama.cpp, which uses ggml, was a breakthrough project released in March 2023 that first made it practical to run models like Meta's LLaMA on consumer devices without specialized NVIDIA GPUs. Hugging Face is a central hub for the AI community, best known for its Transformers library, which has become the de facto standard framework for defining, training, and sharing AI models.
